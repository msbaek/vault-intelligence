## 연재 1편: "자전거는 백과사전으로 배우지 않는다"

### 핵심 메시지
- 점진적 학습의 본질 (자전거, 말 배우기 비유)
- 비싼 강의가 지름길이 아닌 이유
- 의도적 연습의 중요성

### 구조
```
도입 (300자)
- Hook: "DDD 50만원, JPA 30만원... 이것만 들으면 전문가?"
- 질문: 아이가 문법책 읽고 말 배우던가요?

본문 1: 두 가지 배움의 공통점 (400자)
- 자전거: 넘어지면서 배운다
- 말: 말하면서 문법을 체득한다
- 공통점: 점진적, 실행하면서, 지름길 없음

본문 2: 비싼 강의의 한계 (500자)
- 이상호님 사례: "알맹이가 없음"
- DDD/JPA를 강의로만 배울 수 없는 이유
- "어렴풋이 알고 시작하라" - 완벽한 상태 불가

본문 3: 의도적 연습 (400자)
- 10,000시간 신화의 진실 (12%만 설명)
- 진짜 중요한 것: 피드백을 받으며 반복
- Kent Beck: 같은 것을 30번 구현, 28번째에 새 발견

마무리 (200자)
- "건강하게 성장하라. 꼼수 없이, 지름길 없이"
- 다음 편 예고: "그럼 AI는 어떻게 활용해야 하나?"
```

## 연재 2편: "AI는 PT 트레이너다"

### 핵심 메시지
- AI로 결과물 vs AI로 내 근육(역량) 키우기
- 증강 코딩 vs 바이브 코딩
- 24개월 → 9개월 램프업의 비밀

### 구조
```
도입 (250자)
- 1편 연결: "지름길이 없다면, AI는 뭐지?"
- 질문: AI로 80% 결과물, 그게 경쟁력인가?

본문 1: PT 트레이너 비유 (500자)
- 박태웅 인용: "나 대신 운동해 달라 하면 트레이너만 건강"
- 에세이 숙제의 목적: 노벨상 에세이가 아니라 역량
- AI로 결과물 → 모두가 할 수 있음 → 차별화 없음

본문 2: 증강 코딩의 힘 (500자)
- Kent Beck: "생산성이 아닌 학습을 위해 관리하라"
- 바이브 코딩 vs 증강 코딩
- 24개월 → 9개월 램프업 (탐색 공간 축소)
- AI는 끝없이 인내심 있는 튜터

본문 3: AI 활용 실전 (350자)
- 논문 이해에 Claude 활용 경험
- 아티클 요약 slash command로 수십 개 소화
- 가끔은 AI 없이 혼자 해보기 (실력 유지)

마무리 (200자)
- "내가 할 수 있는 능력을 유지하는 것이 중요"
- 다음 편 예고: "그래서 뭘 어떻게 배워야 하나?"
```

## 연재 3편: "조각 지식으로 전문가 되기"

### 핵심 메시지
- 조각 지식 학습 전략
- 자신만의 루틴
- 차별화 포인트 (핵심 도메인)

### 구조
```
도입 (250자)
- 1, 2편 요약: 지름길 없음 + AI는 트레이너
- 핵심 질문: "모두가 AI 쓰면 경쟁력은 어디서?"

본문 1: 조각 지식의 힘 (450자)
- 지식 배가 곡선: 지금 많이 아는 것은 중요치 않음
- 근시일 내 사용할 지식 → 내 것
- 안 쓸 기술 배우면 → 잊어버림 or 구식화
- 매일 만나는 모르는 것을 조각조각

본문 2: 자신만의 루틴 (400자)
- 아침 RSS Feed, Newsletter 분류
- 제목만/서론만/끝까지/튜토리얼 구분
- 글쓰기/발표로 피드백 받기
- Kolb 학습 주기: 경험 → 성찰 → 개념화 → 실험

본문 3: 차별화 포인트 (450자)
- 본질적 복잡성 vs 우발적 복잡성
- 우발적 복잡성(Spring, JPA): AI가 잘함
- 핵심 도메인: GitHub에 공개 안됨 → AI 학습 못함
- Addy Osmani: "AI가 틀릴 때를 아는 것"이 차별화
- 문제 분해 능력이 핵심

마무리 (250자)
- "바다를 막을 수는 없지만 서핑은 배울 수 있다"
- "건강하게 성장하라. 매일 조금씩, 머지않아 전문가"
- 시리즈 마무리
```
