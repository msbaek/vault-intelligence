### 192. GraphQL Mock API 만들기.md (8개 기법)

#### 1. 점진적 구현 전략 (Progressive Implementation Strategy)
- **설명**: 본격적인 구현 전에 Mock API를 먼저 구축하여 인터페이스와 데이터 구조를 검증
- **적용 상황**: 복잡한 API 개발 시작 시, 프론트엔드-백엔드 병렬 개발이 필요할 때

#### 2. 제약 조건 명시 프롬프팅 (Constraint-Based Prompting)
- **설명**: AI에게 코드 생성 시 구체적이고 상세한 제약 조건을 나열하여 원하는 형태의 코드를 정확하게 생성
- **적용 상황**: 특정 아키텍처 패턴을 준수해야 할 때, 프로젝트의 코딩 컨벤션이 엄격할 때
- **예시**: "DTO는 record로 구현", "Controller에 nested class로 구현", "부연 설명 없이 코드만 작성"

#### 3. 입출력 예시 기반 생성 (Example-Driven Generation)
- **설명**: GraphQL 쿼리와 예상 응답 데이터를 함께 제공하여 AI가 정확한 스키마와 구현을 생성하도록 유도
- **적용 상황**: API 스펙이 명확히 정의되어 있을 때, 데이터 구조가 복잡할 때

#### 4. 테스트 우선 요구사항 (Test-First Requirement)
- **설명**: AI에게 코드와 함께 테스트 코드를 동시에 생성하도록 요청하여 검증 가능한 코드를 얻는 기법
- **적용 상황**: TDD/BDD 방법론을 따를 때, 생성된 코드의 즉시 검증이 필요할 때

#### 5. 기술 스택 명시 기법 (Tech Stack Specification)
- **설명**: 사용할 프레임워크, 라이브러리, 언어 버전, 데이터 타입 등을 구체적으로 명시
- **적용 상황**: 멀티 버전 지원이 필요한 기술을 사용할 때, 특정 타입 시스템을 준수해야 할 때

#### 6. 최소 출력 제약 (Minimal Output Constraint)
- **설명**: "부연 설명 없이 코드만 작성"처럼 AI의 장황한 설명을 제거하고 실행 가능한 코드만 받도록 요청
- **적용 상황**: 빠른 프로토타이핑이 필요할 때, 설명보다 구현이 우선일 때

#### 7. 아키텍처 제약 프롬프팅 (Architecture Constraint Prompting)
- **설명**: 클래스 구조, 레이어 분리, 의존성 관계 등 아키텍처 수준의 제약을 명시하여 일관된 코드 구조 유도
- **적용 상황**: 마이크로서비스 아키텍처 구축 시, 레이어드 아키텍처 준수가 필요할 때

#### 8. 어노테이션 화이트리스트 기법 (Annotation Whitelisting)
- **설명**: 사용 가능한 어노테이션을 명시적으로 제한하여 불필요한 의존성이나 설정이 추가되는 것을 방지
- **적용 상황**: 프레임워크 사용이 제한적일 때, 보일러플레이트 코드를 최소화하고 싶을 때

### 195. LLM-Prompts.md (23개 기법)

#### 1. 프롬프트 구성요소 4가지
- **설명**: 효과적인 프롬프트는 지시(instruction), 질문(question), 문맥(context), 예시(few-shot)로 구성
- **적용 상황**: 모든 프롬프트 작성의 기본 원칙

#### 2. 환각(Hallucination) 방지 기법
- **설명**: LLM이 잘못된 정보를 생성하는 것을 방지하는 기법
- **적용 상황**: 정확한 정보가 중요한 업무, 사실 기반 답변 필요 시
- **예시**: "모르는 정보면 모른다고 확실히 대답해주세요"

#### 3. 제로샷(Zero-shot) 프롬프트
- **설명**: 예시 없이 지시만으로 작업을 수행하도록 요청
- **적용 상황**: 단순하고 명확한 작업, 기본적인 질문

#### 4. 퓨샷(Few-shot) 프롬프트
- **설명**: 여러 가지 예시를 제공하여 원하는 형식으로 답변 유도
- **적용 상황**: 복잡한 작업, 일정한 형식의 답변 필요 시

#### 5. 단순하고 명확하게 지시하기
- **설명**: 주관적 표현 대신 구체적이고 측정 가능한 지시 사용
- **적용 상황**: 일관된 답변이 필요한 모든 경우
- **예시**: "2-3개의 문장을 사용해서 고등학생에게 알려주는 것과 같이 설명해줘"

#### 6. 해야 할 것에 집중하기 (부정 지시 회피)
- **설명**: "하지 말아야 할 것"보다 "해야 할 것"으로 지시
- **적용 상황**: 원하는 행동을 명확히 정의할 때

#### 7. 프롬프트 가독성 향상 - 지시와 예시 분리
- **설명**: ## 기호를 사용하여 지시, 예시 등을 명확히 구분
- **적용 상황**: 복잡한 프롬프트 작성 시

#### 8. 프롬프트 리스트 형식으로 나열
- **설명**: 여러 작업을 번호 매긴 리스트로 명확히 구분
- **적용 상황**: 다단계 작업, 복잡한 워크플로우

#### 9. 감정적 미사여구 활용
- **설명**: '제발', '꼭' 등의 미사여구나 상황 설명으로 답변 품질 향상
- **적용 상황**: 중요한 작업, 고품질 답변 필요 시

#### 10. 체인 오브 쏘트(Chain of Thought) - 단계별 사고
- **설명**: "단계별로 생각해서 문제를 풀어줘" 지시로 복잡한 문제 해결
- **적용 상황**: 복잡한 계산, 논리적 추론 필요 시

#### 11. 셀프 크리틱(Self-critique) 요청
- **설명**: AI가 자신의 답변을 검토하도록 요청
- **적용 상황**: 정확성이 중요한 업무

#### 12. 텍스트 요약 기법
- **설명**: 긴 텍스트를 단계적으로 요약
- **적용 상황**: 문서 요약, 정보 추출

#### 13. 정보 추출 기법
- **설명**: 문서나 파일에서 특정 정보만 추출
- **적용 상황**: PDF, Excel 등의 문서 분석

#### 14. 컨텍스트 기반 Q&A
- **설명**: 문맥 제공 후 질문에 답변하도록 지시
- **적용 상황**: 특정 문서 기반 Q&A 시스템

#### 15. 텍스트 분류 기법
- **설명**: 텍스트를 카테고리로 분류 (긍정/부정 등)
- **적용 상황**: 리뷰 분석, 감성 분석

#### 16. 역할 부여(Role Assignment)
- **설명**: GPT에게 특정 역할 부여 (비서, 전문가 등)
- **적용 상황**: 특정 도메인의 전문성 필요 시

#### 17. 출력 형식 지정
- **설명**: 이모지, 테이블, 불렛 포인트 등 원하는 형식 명시
- **적용 상황**: 일관된 형식의 결과물 필요 시

#### 18. 외부 API/도구 통합 (Zapier 등)
- **설명**: GPT를 외부 서비스와 연동하여 자동화
- **적용 상황**: 업무 자동화, 워크플로우 구축

#### 19. 검색 엔진 통합 (SerpAPI)
- **설명**: 실시간 검색 결과를 활용한 정보 수집
- **적용 상황**: 최신 정보 필요 시, 시장 조사

#### 20. 다단계 워크플로우
- **설명**: 여러 작업을 순차적으로 수행하는 복잡한 워크플로우
- **적용 상황**: 뉴스레터 작성, 여행 일정 계획, 이메일-캘린더-노션 통합 등

#### 21. 대화형 정보 수집
- **설명**: 여러 질문을 순차적으로 하여 사용자 정보 수집
- **적용 상황**: 개인화된 추천, 맞춤형 서비스

#### 22. 조건부 검색 및 필터링
- **설명**: 특정 조건(날짜, 위치 등)을 만족하는 정보만 검색
- **적용 상황**: 최신 뉴스 검색, 지역 기반 추천

#### 23. 최적화 알고리즘 적용
- **설명**: 여러 옵션 중 최적의 경로나 선택을 계산
- **적용 상황**: 여행 일정, 경로 최적화

- **설명**: 원격 직무에 맞춘 설득력 있는 커버레터를 작성

- **적용 상황**: 원격 근무 동기 질문 대비

#### 11. 기술 면접 준비
- **설명**: 특정 역할에 대한 기술 면접 준비 - 핵심 주제, 일반적인 기술 질문, 실습 연습 개요 제공
- **적용 상황**: 기술직 원격 면접 준비 시

#### 12. 면접 후 후속 조치
- **설명**: 원격 직무 면접 후 전문적인 후속 이메일 작성

- **적용 상황**: 긴 콘텐츠(영상, 팟캐스트, 기사 등) 요약 시

#### 15. 계획 우선 코딩 프롬프트

- **적용 상황**: 복잡한 구현 전 계획 검토가 필요할 때

### 197. Refactoring.md (12개 기법)

#### 1. 역할 기반 프롬프트 설정 (Role-Based Prompting)
- **설명**: AI에게 특정 전문가 역할을 부여하여 전문성과 일관성 있는 응답을 유도
- **적용 상황**: 코드 리뷰, 리팩터링 요청 시 AI의 응답 품질을 높이고자 할 때
- **예시**: "당신은 Java와 Refactoring 전문가입니다"

#### 2. 구조화된 규칙 제공 (Structured Rule Definition)
- **설명**: AI가 따라야 할 구체적인 규칙들을 계층적으로 명확히 정의
- **적용 상황**: 코드 품질 기준을 명확히 하고 일관된 리팩터링 결과를 얻고자 할 때

#### 3. 단계별 워크플로우 지시 (Step-by-Step Workflow)
- **설명**: 복잡한 작업을 단계별로 분해하고, 각 단계마다 피드백을 요청하도록 설정
- **적용 상황**: 대규모 리팩터링이나 복잡한 코드 개선 작업 시

#### 4. 컨텍스트 기반 참조 제공 (Context-Based Reference)
- **설명**: AI에게 특정 전문 서적이나 방법론을 참조하도록 명시
- **적용 상황**: 업계 표준이나 검증된 방법론에 따른 코드 개선이 필요할 때
- **예시**: Kent Beck의 "TDD By Example", Joshua Kerievsky의 "Refactoring to Patterns"

#### 5. 예제 기반 학습 (Example-Based Learning)
- **설명**: Before/After 예제를 제공하여 AI가 원하는 스타일과 품질을 학습하도록 유도
- **적용 상황**: 특정 코딩 스타일이나 패턴을 AI에게 명확히 전달하고자 할 때

#### 6. DSL 기반 테스트 작성 프롬프트 (DSL-Driven Test Design)
- **설명**: 테스트 코드를 DSL(Domain Specific Language)과 Protocol Driver Layer로 분리하여 가독성과 유지보수성 향상
- **적용 상황**: 테스트 코드의 가독성을 높이고 비즈니스 의도를 명확히 표현하고자 할 때

#### 7. 검증 및 피드백 메커니즘 (Validation and Feedback Loop)
- **설명**: AI에게 자체 검증과 불확실성 표현을 요구하여 신뢰성 향상
- **적용 상황**: AI 응답의 정확성이 중요한 기술적 의사결정 시

#### 8. 패턴 기반 리팩터링 가이드 (Pattern-Based Refactoring)
- **설명**: 구체적인 리팩터링 패턴과 원칙을 명시하여 일관된 코드 개선 유도
- **적용 상황**: 코드 품질을 체계적으로 개선하고자 할 때
- **예시**: Slide Statements, Split Variable, Extract Value Object, Split by Abstraction Layer

#### 9. SoC 기반 설계 원칙 적용 (Separation of Concerns)
- **설명**: 관심사의 분리 원칙을 명시하여 모듈화된 코드 구조 유도
- **적용 상황**: 복잡한 시스템의 모듈 분리나 레이어 아키텍처 설계 시

#### 10. 문서화 요구사항 명시 (Documentation Requirements)
- **설명**: AI에게 다이어그램이나 설명 문서 작성을 요구하여 이해도 향상
- **적용 상황**: 복잡한 로직을 시각화하거나 팀원과 공유할 문서가 필요할 때

#### 11. 스트림 파이프라인 변환 (Stream Pipeline Transformation)
- **설명**: 전통적인 for loop를 함수형 스타일의 stream pipeline으로 변환하도록 지시
- **적용 상황**: Java 코드의 가독성과 함수형 프로그래밍 스타일 적용 시

#### 12. CQS(Command-Query Separation) 원칙 적용
- **설명**: Command 메서드와 Query 메서드를 명확히 분리하도록 AI에게 지시
- **적용 상황**: 부수 효과를 명확히 하고 코드의 예측 가능성을 높이고자 할 때

### 198. Slow Think의 놀라운 효과.md (9개 기법)

#### 1. Slow Think (슬로우 싱크) 프롬프트 기법
- **설명**: Chain of Thought보다 4~10% 더 정확한 결과를 제공하는 고급 프롬프트 기법
- **적용 상황**: 중요한 전략 결정, 복잡한 의사결정, 높은 정확도가 필요한 비즈니스 분석
- **예시**: "Take a deep breath and work through this carefully."

#### 2. Chain of Thought (CoT) - 단계적 사고
- **설명**: 복잡한 문제를 단계별로 명확하게 사고하게 하는 기본 기법
- **적용 상황**: 수학 문제, 로직 설명, 계획 및 전략 설계
- **예시**: "Let's think step by step."

#### 3. 신중함 및 정확성 강조 기법
- **설명**: AI가 서두르지 않고 신중하게 정보를 처리하고 답변하도록 유도
- **적용 상황**: 오류가 잦거나 민감한 정보를 다룰 때

#### 4. 전문가 관점 및 역할 부여
- **설명**: 특정 분야 최고 전문가의 심층적이고 체계적인 사고방식과 지식 체계를 적용하도록 유도
- **적용 상황**: 고난이도 전문분야 질문, 특정 역할의 관점에서 문제 해결 필요 시

#### 5. 불확실성 관리 및 정직성
- **설명**: AI가 자신의 지식 한계를 인지하고, 불확실한 정보에 대해 솔직하게 표현
- **적용 상황**: 정보의 모호성이 존재할 때, AI 환각(hallucination) 우려 시

#### 6. 심층 분석 및 다각적 고려
- **설명**: 단일 관점에서 벗어나 여러 각도에서 문제를 종합적으로 분석
- **적용 상황**: 사회적 이슈 분석, 정책 결정, 복잡한 딜레마 상황

#### 7. Step-Back Prompting (한 걸음 물러서기)
- **설명**: 구체적인 문제에 매몰되기 전에, 관련된 일반 원칙이나 배경 지식을 먼저 활성화
- **적용 상황**: 특정 사례에 대한 판단, 역사적 사건 분석, 새로운 개념 이해

#### 8. 자기 비판 및 개선
- **설명**: AI가 스스로 생성한 답변을 비판적으로 검토하고 개선
- **적용 상황**: 보고서 초안 작성, 주장 개발, 창의적 글쓰기, 코드 생성 후 디버깅

#### 9. 문제 분해 및 구조화
- **설명**: 큰 문제를 작고 다루기 쉬운 부분으로 나누어 체계적으로 접근
- **적용 상황**: 프로젝트 계획, 대규모 시스템 설계, 복잡한 연구 질문 해결
