## 오전: 핵심 구현 (10:00-12:00)

### TDD 사이클 시작

#### Phase 1: Red (실패하는 테스트 작성)

**직접 작성 권장** - 테스트는 의도를 표현하는 것

```java
@Test
void shouldHandleSessionTimeoutGracefully() {
    // Given
    User user = createTestUser();
    Session session = loginService.login(user);

    // When
    simulateTimeout(session, Duration.ofMinutes(30));

    // Then
    assertThat(session.isValid()).isFalse();
    assertThat(loginService.canRefresh(session)).isTrue();
}
```

#### Phase 2: Green (AI가 구현)

**프롬프트:**
```markdown
이 테스트가 통과하도록 구현해줘:

[테스트 코드 붙여넣기]

규칙:
- 기존 테스트를 수정하지 마
- 최소한의 코드로 구현해
- 필요하면 하드코딩으로 시작해도 좋아
```

#### Phase 3: Blue (리팩토링)

**프롬프트:**
```markdown
테스트가 통과하니, 이제 리팩토링을 제안해줘.

다음 관점에서 검토해:
1. 중복 제거
2. 명명 개선
3. 메서드 분리
4. 클린 코드 원칙

각 변경에 대해 이유를 설명해줘.
```

### 구현 팁

1. **작은 단계로 진행** - 한 번에 하나의 테스트
2. **자주 커밋** - 각 Green 단계에서 커밋
3. **AI 제안 검토** - 무조건 수용하지 않기
4. **테스트 실행** - 각 변경 후 전체 테스트

## 오후 초반: 확장 및 통합 (13:00-15:00)

### 1. 기능 확장

**컨텍스트 유지하며 확장:**
```markdown
오전에 구현한 세션 타임아웃 처리를 확장하자.

추가할 기능:
1. 자동 세션 갱신
2. 갱신 실패 시 알림
3. 동시 로그인 처리

각 기능에 대해:
- 테스트 케이스 제안
- 구현 순서 제안
해줘
```

### 2. 통합 작업

**병렬 처리 활용:**
```markdown
다음 작업들을 병렬로 진행해줘:

Task 1: 세션 관리 로직 통합 테스트 작성
Task 2: API 엔드포인트 업데이트
Task 3: 문서화 (README 업데이트)
```

### 3. 의존성 관리

```markdown
새로 추가한 기능들이 기존 코드에 미치는 영향을 분석해줘:

- 영향받는 모듈 목록
- 잠재적 충돌 지점
- 마이그레이션 필요 여부
```

## 오후 후반: 리팩토링 및 리뷰 (15:00-17:00)

### 1. 코드 리팩토링

**Tidy First 접근:**
```markdown
오늘 작성한 코드를 Tidy First 방식으로 정리해줘:

1. Guard Clauses 추가
2. 긴 메서드 분리
3. 변수명 개선
4. 주석 필요한 곳 식별

각 변경은 작게, 테스트가 통과하는 상태 유지해줘.
```

### 2. 셀프 코드 리뷰

**프롬프트:**
```markdown
오늘 변경한 파일들을 리뷰해줘:

관점:
1. 보안 취약점
2. 성능 이슈
3. 에러 처리 누락
4. 테스트 커버리지

각 이슈에 Critical/Major/Minor 우선순위 부여해줘.
```

### 3. PR 준비

```bash
/commit
```

```markdown
오늘 변경사항으로 PR을 준비해줘:

포함할 내용:
1. 변경 요약
2. 테스트 방법
3. 스크린샷 (필요시)
4. 리뷰어 체크리스트
```
