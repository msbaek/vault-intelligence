### 42. AI-Driven Development Lifecycle (AI-DLC) Method Definition

**추출 기법**: 12개

1. **AI-Driven Conversation (AI 주도 대화)**
   - AI가 먼저 대화를 시작하고 주도, 인간은 "승인자" 역할 수행
   - 카테고리: 협업 패턴

2. **Intent-Based Development (의도 기반 개발)**
   - 비즈니스 목표를 "Intent"로 캡슐화하여 AI가 User Stories, AC, 작업 단위로 분해
   - 카테고리: 프롬프트 엔지니어링/워크플로우

3. **AI-Generated Level 1 Plan**
   - AI에게 구현을 위한 워크플로우 계획(Level 1 Plan) 생성 요청
   - 카테고리: 워크플로우/도구 활용

4. **Bolt-Based Rapid Iteration**
   - 시간/일 단위의 빠른 반복 주기(Bolt) 도입
   - 카테고리: 생산성 향상/워크플로우

5. **Domain-Driven Design with AI**
   - AI가 DDD 원칙으로 시스템을 Bounded Context로 자동 분해
   - 카테고리: 코드 품질 개선/도구 활용

6. **Mob Elaboration (협업적 요구사항 정교화)**
   - 팀 전체가 AI와 함께 요구사항을 정교화하는 의식
   - 카테고리: 협업/워크플로우

7. **Mob Construction (팀 전체 AI 협업 개발)**
   - 모든 팀이 AI와 함께 도메인 모델링, 설계, 코드 생성 수행
   - 카테고리: 협업/생산성 향상

8. **AI-Generated Automated Testing**
   - AI가 각 Unit에 대해 실행 가능한 코드와 함께 자동 테스트 생성
   - 카테고리: 코드 품질 개선/도구 활용

9. **AI-Driven Operations Monitoring**
   - AI가 텔레메트리 데이터를 능동적으로 분석하여 패턴 감지, 이상 식별
   - 카테고리: 도구 활용/워크플로우

10. **Incremental Context Enrichment**
    - 각 단계의 아티팩트를 점진적으로 풍부하게 만들어 다음 단계 컨텍스트로 변환
    - 카테고리: 워크플로우/프롬프트 엔지니어링

11. **AI-Generated PRFAQ**
    - AI가 Press Release/FAQ 문서를 자동 생성 (Working Backwards)
    - 카테고리: 도구 활용/생산성 향상

12. **Human-in-the-Loop Validation**
    - AI 생성물에 대해 인간이 검증, 옵션 선택, 결정 확인 수행
    - 카테고리: 협업 패턴/코드 품질

### 50. Backlog.MD and Spec-Driven Development

**추출 기법**: 12개

1. **Spec-Driven AI Development**
   - PRD → 작업 분해 → 구현 계획 → 코드 구현의 단계적 프로세스로 50%→95% 성공률 달성
   - 카테고리: 워크플로우

2. **컨텍스트 윈도우 최적화 (One Task Per Context Window)**
   - 기능을 단일 컨텍스트에 맞는 작은 작업으로 분해
   - 카테고리: 프롬프트 엔지니어링/워크플로우

3. **마크다운 파일을 영구적 컨텍스트로 활용**
   - 마크다운이 AI 에이전트를 위한 영구적 컨텍스트 역할
   - 카테고리: 도구 활용/협업

4. **구현 계획 우선 검토 (Plan First, Code Second)**
   - 코드 작성 전 구현 계획 생성 및 검토가 가장 중요
   - 카테고리: 워크플로우/코드 품질

5. **명확한 수락 기준(Acceptance Criteria) 작성**
   - 측정 가능하고 검증 가능한 체크박스 형식 AC 작성
   - 카테고리: 프롬프트 엔지니어링/협업

6. **에이전트 명령 지속적 개선**
   - 문제 발생 시 에이전트 명령(claude.md, agents.md) 업데이트
   - 카테고리: 워크플로우/도구 활용

7. **환각 주도 개발 (Hallucination-Driven Development)**
   - AI가 환각하는 존재하지 않는 명령이 유용하면 실제로 구현
   - 카테고리: 협업/워크플로우

8. **완료 정의(Definition of Done) 명시적 지정**
   - AI가 조기 완료 선언 방지를 위해 명시적으로 지시
   - 카테고리: 프롬프트 엔지니어링

9. **3단계 검토 체크포인트**
   - (1) 작업 설명 및 AC 검토, (2) 구현 계획 검토, (3) 코드 및 결과 검토
   - 카테고리: 코드 품질/협업

10. **작업별 모델 전문화**
    - 계획에는 계획 모델, 코딩에는 코딩 모델 사용
    - 카테고리: 도구 활용/생산성

11. **가드레일 설정 (테스트, 린팅)**
    - 자동화된 품질 검증 도구로 AI 결과물 검증
    - 카테고리: 코드 품질

12. **Good Enough 수용**
    - 100% 품질 기대 대신 충분히 좋은 결과 수용 후 반복 개선
    - 카테고리: 워크플로우/생산성
